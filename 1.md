# 🔹 1. JDK vs JRE vs JVM

### ✔️ Definitions

* **JVM** → Executes bytecode
* **JRE** → JVM + libraries (to run Java apps)
* **JDK** → JRE + tools (compiler, etc.)

---

### 🔥 Tricky Question

👉 Do we need JDK in production?

✅ Answer:

* ❌ No (only JRE needed to run `.jar`)
* ✔️ JDK needed for build/compile

---

# 🔹 2. .class vs .jar

### ✔️ Difference

* `.class` → bytecode (single file)
* `.jar` → collection of `.class` + metadata

---

### 🔥 Tricky

👉 Can you run `.class` directly?

✔️ Yes:

```
java A
```

👉 `.jar` needs:

```
java -jar app.jar
```

---

# 🔹 3. Compilation Flow

```text
.java → (javac) → .class → (jar) → .jar → (JVM) → run
```

---

### 🔥 Tricky

👉 Does `.jar` contain source code?

❌ No → only bytecode

# 🔹 4. Primitive Data Types

| Type    | Size          | Range       |
| ------- | ------------- | ----------- |
| byte    | 1B            | -128 to 127 |
| short   | 2B            | -32K to 32K |
| int     | 4B            | ~2B         |
| long    | 8B            | huge        |
| float   | 4B            | decimal     |
| double  | 8B            | precise     |
| char    | 2B            | unicode     |
| boolean | JVM dependent |             |

---

### 🔥 Tricky

```java
System.out.println(10/0);
```

❌ ArithmeticException

```java
System.out.println(10.0/0);
```

✅ Infinity

---

# 🔹 5. int vs long (Production)

👉 Use:

* `int` → default
* `long` → when overflow risk

---

### 🔥 Tricky

```java

x++;
System.out.println(x);
```

✅ Output:

```
-2147483648
```

👉 Overflow wraps around

# 🔹 6. String & String Pool

### ✔️ String is a class (immutable)

---

### 🔥 Tricky

```java

String b = "Hello";
System.out.println(a == b);
```

✅ true (same pool)

---

```java
String a = new String("Hello");
String b = "Hello";
System.out.println(a == b);
```

❌ false
