# Complete Java Notes — Abstract Class, Interface, Functional Interface, Object Class, Inner Classes

# 1. Abstract Class in Java

An abstract class is a class that cannot be instantiated directly.

```java
abstract class Animal {
}
```

❌ Invalid:

```java
Animal a = new Animal();
```

---

# Why Abstract Class?

Used when:

* some common behavior exists
* some behavior must be implemented by child classes

Example:

```java
abstract class Animal {

    void eat() {
        System.out.println("Eating");
    }

    abstract void sound();
}
```

Here:

* `eat()` → complete method
* `sound()` → incomplete method

---

# Abstract Method

Method without body.

```java
abstract void sound();
```

Rules:

* ends with `;`
* no implementation

---

# Child Must Implement Abstract Methods

```java
abstract class Animal {
    abstract void sound();
}

class Dog extends Animal {

    void sound() {
        System.out.println("Bark");
    }
}
```

---

# If Child Doesn't Implement?

Then child must also become abstract.

```java
abstract class Dog extends Animal {
}
```

---

# Abstract Class Can Have

✅ Constructors
✅ Variables
✅ Static methods
✅ Final methods
✅ Normal methods
✅ Abstract methods

---

# Constructor in Abstract Class

```java
abstract class Animal {

    Animal() {
        System.out.println("Animal Constructor");
    }
}

class Dog extends Animal {

    Dog() {
        System.out.println("Dog Constructor");
    }
}
```

Output:

```text
Animal Constructor
Dog Constructor
```

---

# Important Rules

---

## Rule 1

If class contains abstract method → class must be abstract.

---

## Rule 2

Abstract class may or may not contain abstract methods.

Valid:

```java
abstract class A {

    void show() {
    }
}
```

---

## Rule 3

Cannot create object of abstract class.

---

# Invalid Combinations

---

## abstract + final

❌ Invalid

```java
abstract final class A {
}
```

Reason:

* abstract → must inherit
* final → cannot inherit

---

## abstract + private method

❌ Invalid

```java
abstract private void test();
```

Reason:
private methods are not overridden.

---

## abstract + static method

❌ Invalid

```java
abstract static void test();
```

Static methods are not overridden.

---

## abstract + final method

❌ Invalid

```java
abstract final void test();
```

Contradiction.

---

# Runtime Polymorphism with Abstract Class

```java
abstract class Animal {
    abstract void sound();
}

class Dog extends Animal {

    void sound() {
        System.out.println("Bark");
    }
}

public class Main {

    public static void main(String[] args) {

        Animal a = new Dog();

        a.sound();
    }
}
```

Output:

```text
Bark
```

---

# Tricky Question

```java
abstract class A {

    A() {
        test();
    }

    abstract void test();
}

class B extends A {

    int x = 10;

    void test() {
        System.out.println(x);
    }
}
```

```java
new B();
```

Output:

```text
0
```

Why?
Because child variables initialize after parent constructor.

# 2. Interface in Java

Interface defines contract/capability.

---

# Interface Methods

By default all methods are:

```java
public abstract
```

So:

```java
interface Animal {
    void sound();
}
```

Actually becomes:

```java
interface Animal {
    public abstract void sound();
}
```

---

# Implementing Interface

```java
interface Animal {
    void sound();
}

class Dog implements Animal {

    public void sound() {
        System.out.println("Bark");
    }
}
```

---

# Why Method Must Be Public?

Because interface methods are public.

Reducing visibility is not allowed.

---

# Interface Variables

All interface variables are automatically:

```java
public static final
```

Example:

```java
interface Test {

    int x = 10;
}
```

Actually:

```java
public static final int x = 10;
```

---

# Java 8 Interface Features

---

## Default Methods

```java
interface A {

    default void show() {
        System.out.println("Hello");
    }
}
```

---

## Static Methods

```java
interface A {

    static void test() {
        System.out.println("Static");
    }
}
```

Called using:

```java
A.test();
```

---

# Java 9 Features

---

## Private Methods

```java
interface A {

    private void helper() {
    }
}
```

---

# Interface vs Abstract Class

| Feature              | Interface | Abstract Class |
| -------------------- | --------- | -------------- |
| Multiple inheritance | ✅         | ❌              |
| Constructors         | ❌         | ✅              |
| Instance variables   | ❌         | ✅              |
| Default methods      | ✅         | ✅              |
| Object state         | ❌         | ✅              |

---

# When to Use Interface?

Use for:

* capabilities
* loose coupling
* APIs
* contracts

Example:

```java
interface Flyable {
    void fly();
}
```

---

# When to Use Abstract Class?

Use for:

* shared state
* common code
* base hierarchy

Example:

```java
abstract class Vehicle {

    int speed;

    void start() {
    }
}
```

---

# Real Collection Framework Usage

Java Collections uses:

* interface for contract
* abstract class for partial implementation

Examples:

* List
* AbstractList
* ArrayList

Hierarchy:

```text
List
  ↓
AbstractList
  ↓
ArrayList
```

# 3. Functional Interface

A functional interface contains exactly ONE abstract method.

Example:

```java
@FunctionalInterface
interface Animal {

    void sound();
}
```

---

# Why Important?

Foundation of:

* Lambda expressions
* Stream API
* Method references

---

# Lambda Example

```java
Animal a = () -> {
    System.out.println("Bark");
};
```

---

# Functional Interface Can Have

✅ default methods
✅ static methods
✅ private methods

Only abstract methods count.

---

# Built-in Functional Interfaces

Package:

```java
java.util.function
```

---

# Predicate

```java
Predicate<Integer> p = n -> n % 2 == 0;
```

Method:

```java
boolean test(T t)
```

---

# Consumer

```java
Consumer<String> c = s -> System.out.println(s);
```

---

# Supplier

```java
Supplier<Integer> s = () -> 100;
```

---

# Function

```java
Function<Integer, Integer> f = n -> n * n;
```

---

# Tricky Question

Is Runnable functional interface?

✅ Yes

Because:

```java
void run();
```

only one abstract method exists.

# 4. Object Class

Every class in Java implicitly extends `Object`.

Example:

```java
class Dog {
}
```

Internally:

```java
class Dog extends Object {
}
```

---

# Abstract Classes Also Extend Object

```java
abstract class Animal {
}
```

Actually:

```java
abstract class Animal extends Object {
}
```

---

# Common Object Methods

Inherited methods:

* `toString()`
* `equals()`
* `hashCode()`
* `getClass()`
* `wait()`
* `notify()`

---

# Upcasting

```java
Object obj = new Dog();
```

Valid because:

```text
Object
   ↓
Dog
```

---

# Important Concept

Reference type controls:

* accessible methods

Object type controls:

* overridden method execution

---

# Example

```java
class Animal {

    void sound() {
        System.out.println("Animal");
    }
}

class Dog extends Animal {

    void sound() {
        System.out.println("Bark");
    }
}

Animal a = new Dog();

a.sound();
```

Output:

```text
Bark
```

Runtime polymorphism.

---

# Tricky Question

```java
Object obj = new Object();

Dog d = (Dog) obj;
```

Compiles but runtime exception:

```text
ClassCastException
```

# 5. Inner Classes

Inner class = class inside another class.

---

# Types of Inner Classes

| Type                  | Description      |
| --------------------- | ---------------- |
| Member Inner Class    | non-static inner |
| Static Nested Class   | static inner     |
| Local Inner Class     | inside method    |
| Anonymous Inner Class | no name          |

---

# 5.1 Member Inner Class

```java
class Outer {

    int x = 10;

    class Inner {

        void show() {
            System.out.println(x);
        }
    }
}
```

---

# Creating Object

```java
Outer o = new Outer();

Outer.Inner i = o.new Inner();
```

---

# Important

Inner class can access private members of outer class.

---

# 5.2 Static Nested Class

```java
class Outer {

    static int x = 10;

    static class Inner {

        void show() {
            System.out.println(x);
        }
    }
}
```

---

# Object Creation

```java
Outer.Inner i = new Outer.Inner();
```

No outer object needed.

---

# Difference

| Feature                   | Member Inner | Static Nested |
| ------------------------- | ------------ | ------------- |
| Needs outer object        | ✅            | ❌             |
| Access non-static members | ✅            | ❌             |

---

# 5.3 Local Inner Class

Class inside method.

```java
class Outer {

    void test() {

        class Inner {

            void show() {
                System.out.println("Hello");
            }
        }
    }
}
```

---

# Access Rules

Can access:

* outer instance variables
* outer static variables
* local variables (only effectively final)

---

# Effectively Final Example

```java
void test() {

    int x = 10;

    class Inner {

        void show() {
            System.out.println(x);
        }
    }
}
```

Valid.

---

# Invalid Example

```java
int x = 10;

x++;

class Inner {
}
```

❌ Error

---

# Why Final Needed?

Local variables live in stack memory.

After method ends they disappear.

Compiler copies value into inner class object.

---

# 5.4 Anonymous Inner Class

Class without name.

---

# Example

```java
abstract class Animal {

    abstract void sound();

Animal a = new Animal() {

    void sound() {
        System.out.println("Bark");
    }
};
```

# Tricky Questions on Inner Classes

---

# Q1 Can inner class have static methods?

❌ Normally no.

Exception:

```java
static final int x = 10;
```

allowed.

---

# Q2 Can outer class access inner private members?

✅ Yes

---

# Q3 Can local inner class access outer variables?

✅ Yes

| Variable Type     | Accessible?              |
| ----------------- | ------------------------ |
| Instance variable | ✅                        |
| Static variable   | ✅                        |
| Local variable    | ✅ effectively final only |

---

# Q4 Why local variables must be effectively final?

Because:

* local variables live in stack
* inner class object lives in heap

Different lifetimes.
